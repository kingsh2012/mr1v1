# CS 1.6 1v1 对战平台 — 服务端建设阶段目标

## 背景

基于 rehlds 开源生态（ReHLDS + ReGameDLL_CS + Metamod-R + AMXModX + ReAPI）搭建一套供 1v1 对战平台使用的 CS 1.6 竞技服务端。以 PROCS.PRO 为架构参考，重新立项开发。目标是稳定、可控、可采集数据：平台后端能远程下发指令、读取比赛数据，服务器本身能按需弹起和销毁。

---

## 阶段零：服务端设计

**目标：** 在动手之前把架构设计清楚，输出设计文档作为后续实施依据。

- **1v1 插件功能边界定义**
  - 确定哪些功能做、哪些不做
  - 与 PROCS.PRO 的 `procs_cmps.sma` 功能对比，明确裁剪点
- **插件列表与职责划分**
  - 参考 PROCS.PRO 插件体系，确定新服务端需要哪些插件
  - 明确各插件之间的依赖关系（native / forward）
- **比赛状态机设计**
  - ready → 刀局 → 换边 → 上半场 → 下半场 → 结束
  - 每个状态的触发条件、转移逻辑、超时处理
- **数据上报字段设计**
  - 确定采集哪些事件（比赛开始/结束、回合、击中、死亡等）
  - 定义每个事件的 JSON 数据结构
- **服务器配置体系设计**
  - cfg 文件结构（server.cfg、match.cfg 等）
  - 启动参数与环境变量注入方式

**产出物：** 服务端设计文档

---

## 阶段一：单实例跑通（基础环境）

**目标：** 基于设计文档，在 Linux 机器上部署一个可连接的 ReHLDS 实例。

- 安装 ReHLDS 引擎（替代原版 HLDS）
- 安装 ReGameDLL_CS（替代原版 cs.so）
- 安装 Metamod-R + AMXModX 1.10 + ReAPI
- 验证：能用 CS 1.6 客户端连上，基础地图正常运行
- 配置 RCON 密码，验证远程控制可用

**产出物：** 可重复执行的部署脚本

---

## 阶段二：1v1 比赛插件开发

**目标：** 按设计文档实现一场完整 1v1 竞技对局的全生命周期控制。

- 开发核心比赛插件（.sma），覆盖完整状态机：
  - 热身阶段（warmup）：双方 ready 后开始
  - 刀局决定换边
  - 正式局：MR15，强制买枪配置
  - 比赛结束后触发结果上报
- 开发数据采集插件，按设计字段上报事件到 gateway
- 配置竞技 cfg（无友伤、buy time、round time、128tick 等）

**产出物：** 可用的 `.sma` 插件 + `match.cfg`

---

## 阶段三：平台控制接口

**目标：** 平台后端能远程控制服务器的对局生命周期。

- 封装 RCON 客户端，支持：
  - 创建对局（加载 match cfg）
  - 强制结束对局
- 实现服务器 → 平台的事件上报（gateway 接收）
- 联调验证完整数据链路：插件上报 → gateway → MQTT → consumer → DB

**产出物：** RCON 控制库 + 数据链路联调验证

---

## 阶段四：容器化单实例

**目标：** 将单个 CS 1.6 服务器封装为 Docker 镜像，支持参数化启动。

- 编写 Dockerfile（基于 ubuntu:20.04，含 32-bit libs）
- 启动参数通过环境变量注入：`PORT`、`RCON_PASSWORD`、`MAP`、`MATCH_ID`
- 健康检查：UDP 探测端口
- 地图文件单独挂载，控制镜像体积

**产出物：** `Dockerfile` + `docker-compose.yml`（单实例验证用）

---

## 阶段五：多实例调度层

**目标：** 支持动态按需开/关房间，对接平台后端。

- 实现轻量调度服务（Go），提供 HTTP API：
  - `POST /rooms` — 创建房间（拉起容器，分配端口）
  - `DELETE /rooms/:id` — 销毁房间
  - `GET /rooms/:id/status` — 查询房间状态
- 端口池管理
- 容器生命周期管理（Docker SDK）

**产出物：** 调度服务代码 + API 文档

---

## 阶段六：反作弊 & 稳定性加固

**目标：** 生产可用前的最后加固。

- 集成 ReChecker（客户端文件完整性校验）
- 集成 ReUnion（支持非 Steam 客户端，按需）
- 压测：单机并发 N 个房间，监控 CPU/内存/网络
- 异常处理：服务器崩溃自动重启
- 日志集中收集

**产出物：** 压测报告 + 生产部署 checklist

---
